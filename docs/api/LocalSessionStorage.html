LocalSessionStorage

LocalSessionStorage()

LocalSessionStorage — key-value persistence backed by `localStorage`. All keys are automatically prefixed to avoid collisions with other libraries. Values are JSON-serialized so any serializable type can be stored directly. Designed for subclassing: override the core methods (`setItem`, `getItem`, `removeItem`, `clear`, `keys`) to swap in a server-side backend while keeping the same public API.

Examples
// Store and retrieve a value
LocalSessionStorage.setItem('sidebar.collapsed', true)
LocalSessionStorage.getItem('sidebar.collapsed', false) // true
// Subclass with a different prefix
class AppStorage extends LocalSessionStorage {
    static prefix = 'myapp.'
}
// Subclass with a server backend
class ServerStorage extends LocalSessionStorage {
    static async setItem(key, value) {
        await fetch('/api/storage', {
            method: 'POST',
            body: JSON.stringify({ key: this.prefix + key, value })
        })
    }
}
