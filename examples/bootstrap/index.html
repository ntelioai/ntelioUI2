Badges, Buttons, Spinners, Alerts etc.

Essential Bootstrap 5.3 constructs you can use directly in ntelioUI2 widget templates.

Why no Button widget? No Badge widget?

ntelioUI2 does not wrap basic HTML/CSS elements in Widget subclasses. Bootstrap 5.3 is loaded globally and provides all standard UI primitives out of the box — buttons, badges, alerts, spinners, cards, form inputs, and more.

Rule of thumb: If Bootstrap already does it, use Bootstrap directly in your templates. Only create a Widget when you need lifecycle management, state, events, or complex behavior.

When do you need a Widget?

When your component has state, timers, events, or cleanup logic — things Bootstrap alone can't manage.

This ActivityMonitor is a Widget. It uses Bootstrap cards, badges, progress bars, list groups, and buttons for its look — but it needs Widget because of:

  • Lifecycle — destroy() clears the interval timer so it doesn't leak
  • State — tracks event count, running/paused status across methods
  • Events — emits started, stopped, complete so parents can react
  • API — exposes start(), stop(), reset() for external control

Click Create Widget, then Start. Events appear in the log and console.

import { Widget } from '../../core/Widget.js' export class ActivityMonitor extends Widget { async init() { this.find('.nui-btn-start').on('click', () => this.start()) } start() { this._interval = setInterval(() => this._tick(), this._speed) this.emit('started') // Custom event → parents can listen } _tick() { this._count++ // Internal state this.find('.nui-progress').css('width', pct) // Update Bootstrap element this.emit('event', { count, level, message }) } destroy() { clearInterval(this._interval) // Lifecycle cleanup! super.destroy() // Widget removes DOM, unbinds events } }
